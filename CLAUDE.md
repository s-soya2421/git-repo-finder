# CLAUDE.md

## 1. 位置づけ

このファイルは Claude Code 向けの補助指示である。
このリポジトリでは **`AGENTS.md` を正本** とする。

**Claude は実装着手前に必ず `AGENTS.md` を Read ツールで読むこと。**

このファイルには Claude Code 固有の注意点と、重要ルールの要約を記載する。
`AGENTS.md` と矛盾がある場合は `AGENTS.md` を優先する。

---

## 2. Claude Code 固有の指示

### ツールの使い分け
- ファイル読み取りには `Read` ツールを使う（`cat` や `head` ではなく）
- ファイル編集には `Edit` ツールを使う（`sed` や `awk` ではなく）
- ファイル検索には `Glob` ツールを使う（`find` ではなく）
- コンテンツ検索には `Grep` ツールを使う（`grep` や `rg` ではなく）
- `Bash` はビルド、テスト実行、git 操作など、専用ツールがない場合に使う

### 実装前の確認手順
1. `AGENTS.md` を Read する
2. 変更対象に関連する `docs/` 配下のドキュメントを Read する
3. 変更対象の既存コードを Read する
4. `AGENTS.md` セクション7 の作業プロトコルに沿って進める

### Server Component の徹底
Claude は `'use client'` を付ける前に、本当に Client Component が必要か再確認すること。
判断基準は `AGENTS.md` セクション6 に従う。

---

## 3. プロダクト価値の要約

Claude はこのプロダクトを単なる GitHub 検索 UI として扱わないこと。
目的は **候補を見つけること** ではなく、**候補を比較し、短時間で判断できること** にある。

実装・提案・レビューでは常に次を意識すること。

- 一覧は比較画面である
- 詳細は最終判断支援画面である
- URL が検索状態の正本である
- GitHub API はサーバー側から扱う
- Server Component を基本にする
- Client Component は必要最小限にする
- 全件閲覧より上位候補比較を優先する
- 過剰な抽象化を避ける

---

## 4. 実装ルールの要約

以下は `AGENTS.md` セクション8 の要約である。詳細は正本を参照すること。

### 絶対に守ること
- GitHub API を UI から直接呼ばない
- トークンをクライアントへ露出しない
- API の生フィールドを UI に直接ばら撒かない
- 既存の命名規則と責務分離に合わせる
- `'use client'` を必要最小限にとどめる
- 純粋関数として切り出せる処理は切り出す

### 避けること
- 大規模なファイル移動
- 意味の薄い共通化
- 依頼と無関係な最適化
- テスト失敗やビルド失敗を黙って放置すること
